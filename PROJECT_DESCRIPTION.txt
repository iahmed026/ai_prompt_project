PROJECT OVERVIEW
================================================================================
The AI Prompt Generator is a full-stack web application that helps users create, 
optimize, and manage AI prompts. The system combines a FastAPI backend with a 
static frontend to provide an intuitive interface for generating high-quality 
prompts tailored to specific niches and use cases.

The application uses OpenRouter API for LLM integration, allowing users to generate 
optimized prompts with support for mock LLM responses for testing and development.


TECHNOLOGY STACK
================================================================================

BACKEND:
  - Framework: FastAPI (v0.128.8) - Modern Python web framework with built-in async support
  - Server: Uvicorn (v0.39.0) - ASGI web server
  - Database: SQLAlchemy (v2.0.49) - ORM for database operations
  - Database Driver: SQLite (local development) / PostgreSQL (production)
  - Database Migrations: Alembic (v1.16.5)
  - Validation: Pydantic (v2.13.4) - Data validation and serialization
  - Authentication: Python-jose (v3.5.0) with bcrypt (v5.0.0)
  - Caching: Cachetools (v6.2.6)
  - HTTP Client: Httpx (v0.28.1) - For external API calls
  - Environment: Python-dotenv (v1.2.1) - Environment configuration
  - Logging: Loguru (v0.7.3)

FRONTEND:
  - HTML5 with semantic markup
  - CSS3 with modern styling
  - Vanilla JavaScript (no framework dependencies for simplicity)
  - Google Fonts (Inter font family)
  - Lucide Icons for UI elements

API ENDPOINTS
================================================================================

VERSION 1 (/api/v1):
  GET  /ui-schema/                              # Get all UI schemas
  GET  /ui-schema/niches/{niche_id}             # Get niche-specific schema
  POST /ui-schema/niches/{niche_id}/add-option  # Add schema option
  POST /generate                                 # Generate optimized prompt
  GET  /history/                                 # Get prompt history
  GET  /health                                   # Health check

AUTHENTICATION (/api/auth):
  POST /login                                    # User login
  POST /register                                 # User registration
  POST /refresh                                  # Refresh token

PROMPTS (/api/prompts):
  POST /                                         # Create new prompt
  GET  /{prompt_id}                             # Get prompt details
  GET  /                                         # List user prompts

HISTORY (/api/history):
  GET  /                                         # Get generation history
  GET  /{id}                                    # Get specific history item

NICHES (/api/niches):
  GET  /                                         # List all niches
  POST /                                         # Create new niche
  GET  /{niche_id}                              # Get niche details

2. User (inferred from project structure)
   - id: Integer (Primary Key)
   - email: String (Unique)
   - username: String (Unique)
   - hashed_password: String
   - created_at: DateTime

3. Niche (inferred from project structure)
   - id: String (Primary Key)
   - label: String
   - description: Text (optional)
   - schema: JSON
   - created_at: DateTime


CONFIGURATION
================================================================================

Environment Variables (.env):
  - DATABASE_URL: SQLite or PostgreSQL connection string
  - OPENROUTER_API_KEY: API key for OpenRouter service
  - OPENROUTER_MODEL: LLM model to use (default: gpt-3.5-turbo or similar)
  - USE_MOCK_LLM: Boolean (true for testing without API calls)
  - APP_NAME: Application name
  - ALLOWED_ORIGINS: CORS allowed origins
  - JWT_SECRET: Secret key for JWT token generation
  - JWT_ALGORITHM: Algorithm for JWT encoding (default: HS256)
  - FRONTEND_DIR: Path to frontend files

Default Configuration:
  - Database: SQLite at backend/prompt_generator.db
  - Server: http://127.0.0.1:8000
  - CORS Origins: localhost:5500, localhost:8000, 127.0.0.1:5500, 127.0.0.1:8000

WORKFLOW
================================================================================

1. USER INITIATES REQUEST
   ├─ User selects a niche
   ├─ User fills out dynamic form based on niche schema
   └─ User submits prompt generation request

2. BACKEND PROCESSES REQUEST
   ├─ Validate user input
   ├─ Build prompt template from inputs
   ├─ Check cache for similar prompts
   ├─ If not cached, call LLM to optimize
   └─ Store result in prompt history

3. RESPONSE TO USER
   ├─ Return optimized prompt
   ├─ Display in UI
   └─ Add to history

4. HISTORY MANAGEMENT
   ├─ User can view all past prompts
   ├─ User can review optimization details
   └─ User can reuse or modify prompts


RUNNING THE APPLICATION
================================================================================

LOCAL SETUP:
  1. Navigate to project directory:
     cd ~/Desktop/ai_prompt_project

  2. Create virtual environment:
     /usr/bin/python3 -m venv venv

  3. Activate virtual environment:
     source venv/bin/activate

  4. Upgrade pip and install dependencies:
     python -m pip install --upgrade pip setuptools wheel
     cd backend
     pip install -r requirements.txt

  5. Run the application:
     python -m uvicorn app.main:app --reload

  6. Access the application:
     - Frontend: http://127.0.0.1:8000
     - Swagger Docs: http://127.0.0.1:8000/docs

DOCKER DEPLOYMENT:
  - docker-compose.yml available for containerized deployment
  - Supports full stack deployment with backend and database
